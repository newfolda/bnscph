# Project Goal

Rebuild the supplied reference website as a clean, responsive frontend.

## Technology

- Next.js App Router
- React
- TypeScript
- Tailwind CSS
- Lucide React icons
- Frontend only for now
- No backend integration yet

## Reference Files

- reference/head-original.txt
- reference/body-original.txt

Treat all files inside reference/ as read-only source material.

## Working Rules

1. Do not modify files inside reference/.
2. Do not copy analytics, tracking scripts, CSRF tokens, cookies, or backend code.
3. Do not use jQuery.
4. Do not copy production API endpoints.
5. Do not add backend functionality.
6. Use reusable React components.
7. Implement mobile-first responsive behavior.
8. Preserve unrelated code when making targeted changes.
9. Do not redesign the reference page unless explicitly requested.
10. Run npm run lint after each major task.
11. Run npm run build after each major phase.
12. Before editing, state which files will be changed.
13. Make the smallest necessary change.
14. Do not install dependencies without explaining why.

## Design Priorities

1. Layout and spacing
2. Typography
3. Colors
4. Responsive behavior
5. Interactions
6. Animation

## Required Viewports

- 390px mobile
- 768px tablet
- 1024px laptop
- 1440px desktop

## Accessibility

- Semantic HTML
- Keyboard-accessible navigation
- Visible focus states
- Accessible labels
- Correct heading hierarchy
- Respect prefers-reduced-motion

## Asset Rules

- Temporary placeholders are allowed.
- Do not permanently hotlink production assets.
- Keep images inside public/images when local assets are available.
